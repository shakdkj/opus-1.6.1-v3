# 算法设计文档：SILK 增益矩阵隐写

## 1. 算法原理

### 1.1 载体选择

Opus 在 SILK 模式下每 20ms 帧产生 4 个子帧增益索引 `GainsIndices[0..3]`，每个取值 0~47（条件编码）或 0~63（独立编码）。4 个增益的修改空间构成 5^4 = 625 候选（每个增益最多 ±2）。

### 1.2 符号映射

将 4 个增益的加权和取模 64，得到 6-bit 符号：

```
symbol = (g0×1 + g1×3 + g2×9 + g3×27) & 63
```

权重 1、3、9、27 满足：任意两个不同符号对应不同的 6-bit 值（单射），保证 6 bit 可无冲突地映射到 64 个符号。

### 1.3 嵌入

对目标 6-bit 符号 target：
1. 遍历 625 候选 (d0,d1,d2,d3)，d ∈ [-2, 2]
2. 筛选 symbol(candidate) == target 的候选
3. 选择 L1 代价 |d0|+|d1|+|d2|+|d3| 最小的候选
4. 若 L1 打平 → L2 代价 d0²+d1²+d2²+d3²
5. 再打平 → 字典序 (c0, c1, c2, c3)
6. 修改增益索引为选中的候选值

### 1.4 提取

```
symbol = (g0×1 + g1×3 + g2×9 + g3×27) & 63
```

直接计算 6-bit 符号并输出，无需任何搜索。

## 2. 预算累积器

问题：目标嵌入率 300 bps，每帧 20ms，但每帧只能嵌入整数 bit（5 或 6）。

解决：预算累积器 `budget_acc` 以 bit×sample_rate 为单位累加：

```
每帧: budget_acc += bps × frame_samples
      allowed = budget_acc / sample_rate   // 本帧可嵌入 bit 数
      budget_acc -= allowed × sample_rate  // 扣除已分配
      if allowed > 6: allowed = 6          // 上限
```

对 300 bps、48000 Hz、960 采样/帧：
- budget_acc += 300×960 = 288000
- allowed = 288000/48000 = 6
- budget_acc -= 6×48000 = 288000
- 归零 → 下一帧同样过程

每帧恰好分配 6 bit，编解码端完全同步。

## 3. 丢包对抗 (seq + XOR-3 FEC)

### 3.1 帧序号 (seq)

每帧的 6 bit 拆为 `[seq:2bit][payload:4bit]`：
- seq 模 4 循环递增（0→1→2→3→0→...）
- 解码端检测 seq 不连续 → 自动补零占位
- 占用 2/6 = 33% 容量

### 3.2 XOR-3 前向纠错

将 4-bit 载荷 nibble 每 2 个一组，计算 XOR 校验 nibble：

```
block: [d0:4bit][d1:4bit][p:4bit]  where p = d0 XOR d1
```

任意 1 个 nibble 丢失可通过 XOR 恢复：
- d0 丢 → d0 = d1 XOR p
- d1 丢 → d1 = d0 XOR p
- p 丢 → 无需恢复（p 不是数据）

开销：每 2 数据 nibble 产出 1 校验 nibble，33% 开销。

### 3.3 与 Opus 内部机制的兼容

- **PLC 模式**：`lostFlag == FLAG_DECODE_NORMAL` 不成立 → 跳过隐写提取，不产生垃圾比特
- **LBRR 恢复帧**：正常提取隐写（LBRR 帧的内容与原始帧相近，增益索引通常一致）

## 4. 嵌入率与 PSNR（6 kbps VBR, 100 样本）

| 嵌入率 | bit/帧 | PSNR | 容量 | bit_acc |
|--------|--------|------|------|---------|
| 300 bps | 6 | 46.3 dB | 37.5 B/s | 100% |
| 150 bps | ~3 | 48.5 dB | 18.8 B/s | 100% |
| 100 bps | ~2 | 49.4 dB | 12.6 B/s | 100% |
| 300 bps + 15%丢包 | 6 (seq) | 30.6 dB | 16.7 B/s | 96.4% |

## 5. 时间复杂度

| 操作 | 复杂度 | 每帧耗时 (估算) |
|------|--------|----------------|
| 嵌入（矩阵搜索） | O(625) = 常数 | < 10 μs |
| 提取 | O(1) | < 1 μs |
| FEC 编码 | O(N) | < 10 μs (N ≤ 256B) |
| FEC 解码 | O(N) | < 10 μs |

整体编解码时延增量 < 0.1 ms，远小于 Opus 自身体积（~2-5 ms 编码 + ~1 ms 解码）。
