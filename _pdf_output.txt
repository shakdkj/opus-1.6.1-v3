Abstract—Voice over Internet Protocol (VoIP) provides a 
crucial 
cover 
object 
for 
secure 
real-time 
streaming 
steganography. Conventional VoIP steganography methods often 
fall short of Kerckhoffs' principle, rendering them vulnerable 
when the steganographic algorithm is known. This study 
introduces a novel steganographic scheme that leverages the 
complex transformation space of Rubik's Cube and hyperchaos 
theory to enhance security and embedding capacity in covert 
VoIP communications. By generating a random Rubik's Cube 
matrix state using hyperchaotic sequences, the scheme ensures a 
high degree of randomness and unpredictability, thereby 
providing 
robust 
confidentiality. 
Theoretical 
analysis 
of 
steganographic security proves that the proposed method for 
covert VoIP communications is secure, as it adheres to 
Kerckhoffs' principle, utilises the robustness of hyperchaotic 
sequences, and resists exhaustive attacks. The effectiveness of the 
steganographic algorithm was examined through performance 
measurements, revealing minimal impact on real-time VoIP 
communications in terms of imperceptibility, speech quality, and 
signal distortion. Security tests conducted using the ANOVA 
method further validate the scheme's resilience against 
steganalysis, demonstrating that it is statistically undetectable 
with a larger embedding capacity of 16 kbps.

Internet and within the Internet of Things (IoT), using it as a 
carrier for covert communication has garnered significant 
interest from researchers. Information hiding techniques based 
on static carriers have become well-established, including text, 
image, audio and video steganography [1]. Currently,

robust speech steganography algorithm designed for use with

III. PROPOSED HYPERCHAOTIC RUBIK’S CUBE-BASED COVERT 
VOIP COMMUNICATIONS SCHEME  
The proposed covert VoIP communications scheme, based on 
hyperchaotic 
Rubik’s 
cube 
steganography, 
combines 
steganography and cryptography to ensure information security. 
The covert VoIP communications are realized by embedding 
encrypted secret message into VoIP streams. The random 
sequence generated by the Rossler hyperchaotic system controls 
the rotation of the third-order Rubik's Cube, and then embeds 
secret information into the VoIP stream carrier based on the 
three-dimensional coordinate matrix information corresponding 
to the 64 points in the Rubik's Cube. 
VoIP communication consists of a signaling phase and a 
session phase. During the signaling phase, a connection is set up 
between the parties using the SIP signaling protocol. In the 
session phase, real-time VoIP communication is facilitated by 
media protocols, while the IP protocol handles the voice 
transmission over IP networks. To ensure low latency and 
minimal packet loss, the system uses the connectionless UDP 
instead of the TCP. 
In order not to affect connection establishment between 
communication parties using SIP signaling protocol in the 
signaling phase, the initial parameters of hyperchaotic sequences 
are securely transmitted between communicating parties during 
the session phase after establishing a connection in this study. A 
dynamic key-distribution algorithm proposed in our previous 
study [32] is adopted in this study for distributing initial 
parameters. The proposed covert VoIP communications model is 
depicted in Figure 1. 
 
 
Fig. 1. Model of covert VoIP Communications. 
 
In the proposed scheme, both communicating parties utilize 
the same initial parameters to generate a random walk 
sequence using the Rossler hyperchaotic system. Subsequently, 
the generated hyperchaotic random sequence is used to control 
the rotation of a third-order Rubik's cube. The resulting cube 
is then placed into a three-dimensional coordinate system, 
yielding a matrix with 64 randomly distributed points from the

C. Embedding and Extraction using Rubik's Cube Matrix 
Steganography 
Place the Rossler hyperchaotic magic cube state into a three-
dimensional coordinate system, resulting in a matrix with 64 
points M (x, y, z) (0 ≤𝑀(𝑥, 𝑦, 𝑧) ≤63) randomly distributed 
within the magic cube (as shown in Figure 4), with each point's 
horizontal, vertical, and depth coordinates ranging from 0 to 3.  
1) Embedding Process 
First, the message to be hidden is encrypted using the AES 
algorithm. The encrypted data is then segmented and 
embedded into VoIP stream packets. In this system, each data 
packet is sized at 4096 bytes. The length of the secret message 
(LoM) is embedded in the first 16 bytes of the initial VoIP 
data packet, while the rest of the first packet, along with 
subsequent  packets, is used for the secret message itself. 
The encrypted secret information is grouped in sets of six 
binary digits. Each set of six binary digits from the 
confidential information can be converted into a decimal 
number (range 0-63), which has a unique coordinate 
correspondence within a 4*4*4 three-dimensional matrix.  
The coordinates corresponding to each group of secret 
message (six binary digits per group) are embedded into last 
two bits of adjacent samples in VoIP stream at three sampling 
points sequentially. The process is as follows: 
a) Select coordinates; iterate through the pre-defined 64 
coordinates (xi, yi, zi);  
b) Embed LoM into the initial VoIP data packet  and set 
length = LoM;  
c) Obtain sampling points; take three adjacent speech 
sampling points C1, C2, C3 from the carrier stream; 
d) Take out 6-bit binary data s1s2s3s4s5s6  from the 
encrypted secret data, calculate the corresponding 
decimal number s (range 0-63), map it to the 
coordinate M （b1b2, b3b4, b5b6 ） in a three-
dimensional hyperchaotic cube matrix.  
e) Embed secret data; embed these 6 bits of data into the 
last two bits of each of the adjacent three audio 
sampling points:  
𝑆1 = (𝐶1&0𝑥𝐹𝐶) ∣𝑏1𝑏2, 
𝑆2 = (𝐶2&0𝑥𝐹𝐶) ∣𝑏3𝑏4, 
𝑆3 = (𝐶3&0𝑥𝐹𝐶) ∣𝑏5𝑏6, 
where “&” is bitwise AND operation, 0xFC is a mask 
used to clear out the last two bits, “|” is bitwise OR 
operation used for embedding data. 
f) 
length = length − 6.  
g) Repeat steps (3)-(6) until length = 0.

IV. SECURITY ANALYSIS 
This section provides a theoretical analysis of the security of 
the 
proposed 
hyperchaotic 
Rubik’s 
Cube-based 
steganographic algorithm for covert communications over 
VoIP. 
A. Kerckhoffs's Principle 
The principle of Kerckhoffs, named after Auguste 
Kerckhoffs in the 19th century, states that even if all details of 
a cryptographic system are known to an adversary, as long as 
the key remains secret, the system should still be secure. 
Currently, most traditional VoIP steganography methods do 
not meet this principle and are vulnerable to attacks when the 
steganographic algorithm is not kept confidential. 
In the proposed algorithm in this paper, the 64 points of a 
Rubik's cube are controlled by a hyperchaotic sequence to 
provide high complexity and randomness. Even if an attacker 
knows 
the 
hyperchaotic Rubik's 
cube 
steganographic 
algorithm, without knowledge of the initial parameters of the 
hyperchaotic system, they cannot obtain its state. With over 
4.32×10^19 possible states for a Rubik's cube, it is extremely 
difficult for current computing power to crack such a system 
in real-time VoIP communication. The key to this algorithm 
lies in how to embed secret message into the sampling points 
of VoIP streams. Even if an attacker knows this mechanism, 
they still cannot access confidential information as long as the 
key (the specific state of the Rubik's cube and its 
corresponding three-dimensional coordinates) remains secret. 
The state of the Rubik's cube can be generated using a 
hyperchaotic sequence which only requires one seed (initial 
parameter) for generation. This eliminates the need to transmit 
or remember the entire distribution of the Rubik's cube; only 
transmitting one seed value is necessary. Additionally, this 
algorithm can be applied to any VoIP streams with high 
applicability; it is easy to operate and does not require 
complex 
calculations 
but 
mainly 
relies 
on 
software 
implementation without additional hardware costs. Therefore, 
this algorithm meets all requirements set forth by Kerckhoffs’s 
Principle. 
 
B. Security of Hyperchaotic 
When employing hyperchaotic systems in steganography 
algorithms, it is essential to use secure hyperchaotic systems 
to ensure the algorithm's security. Due to the information loss 
during iteration in hyperchaotic systems, the information 
entropy of the hyperchaotic sequence gradually approaches 
zero, making it impossible to correctly predict the long-term 
behavior of the 
hyperchaotic sequence. In addition, 
hyperchaotic synchronization is crucial for achieving correct 
decryption of information. Since computer systems can 
accurately reproduce all initial states of a super-chaotic 
iterative 
system, 
a 
synchronization 
algorithm 
for 
a 
hyperchaotic iterative system can be implemented using the 
concept of computer process synchronization, where startup 
parameters and synchronization strategies serve as keys.  
When parameters a = 0.25, b = 3.0, c = 0.05 and d = 0.5 are 
used in Rossler's hyperchaotic system, its Lyapunov

B. Statistical Undetectability Analysis 
Steganographic communications seek to hide secret 
messages within ordinary transmissions, such as VoIP. The 
security of these systems is evaluated based on statistical 
undetectability, which measures how difficult it is to detect 
hidden messages within cover streams. A secure covert 
communication system should be statistically undetectable, 
ensuring that both human perception and computational 
methods cannot differentiate between cover streams and stego 
streams.  
ANOVA was used in this study to evaluate the security of 
the proposed hyperchaotic Rubik’s cube-based steganographic 
algorithm. ANOVA, which stands for analysis of variance, is 
used to test for differences in means between two or more 
groups. It is commonly used to assess whether there are 
significant differences between groups based on a categorical 
variable. ANOVA can be used to test for differences between 
groups, and to determine if there is an interaction between 
groups.  
ANOVA relies on three main assumptions: (1) Homogeneity 
of variance; (2) The population follows normal distribution; (3) 
Each sampling is independent.

message were tested by using ANOVA in this study, and 
each sample containing 64,000 data points. The null 
hypothesis stated no difference between VoIP stream samples, 
while the alternative hypothesis posited a significant 
difference. Steps in the ANOVA Test are as follows: 
Step 1: Calculate the mean and variance of each sample. 
Step 2: Calculate the grand mean.

Step 4: Calculate the degrees of freedom (df) for the F-
distribution. 
Step 5: Calculate the mean square values (MSB and MSW). 
Step 6: Calculate the F-statistic. 
Step 7: Determine the p-value and compare it to the 
significance level.
